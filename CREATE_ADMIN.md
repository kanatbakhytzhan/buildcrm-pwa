# Создание первого администратора

Фронтенд **не создаёт** пользователей — только бэкенд и БД. Чтобы зайти в админку, нужно один раз создать админа на бэкенде.

## Данные для входа (после создания)

| Поле     | Значение                  |
|----------|---------------------------|
| Email    | `kana.bahytzhan@gmail.com` |
| Пароль   | `Kanaezz15!`              |

После первого входа рекомендуется сменить пароль в профиле.

---

## Как создать админа на бэкенде

### Вариант 1: Прямой SQL (если есть доступ к БД)

Структура таблицы у каждого бэкенда своя. Пароль **обязательно** хранить в виде хеша (bcrypt, argon2 и т.п.), не в открытом виде.

Пример (подставьте свою таблицу и генерацию хеша):

```sql
-- Пример для PostgreSQL. Имя таблицы и полей замените на свои.
-- Пароль нужно сгенерировать через ваш бэкенд (например bcrypt.hash('Kanaezz15!')).

-- INSERT INTO users (email, password_hash, is_admin, is_active, created_at)
-- VALUES (
--   'kana.bahytzhan@gmail.com',
--   '<сюда_хеш_пароля_из_бэкенда>',
--   true,
--   true,
--   NOW()
-- );
```

Как получить хеш:

- **Python (FastAPI/Django):** `bcrypt.hashpw(b"Kanaezz15!", bcrypt.gensalt()).decode()` или аналог в вашем фреймворке.
- **Node:** `require('bcrypt').hashSync('Kanaezz15!', 10)`.

Подставьте полученный хеш в `password_hash` и выполните `INSERT`.

### Вариант 2: Скрипт/команда бэкенда

Если в проекте бэкенда есть команда или скрипт типа «создать первого админа» или «seed», запустите его и укажите:

- email: `kana.bahytzhan@gmail.com`
- password: `Kanaezz15!`
- is_admin: `true`

### Вариант 3: Временный эндпоинт (только для разработки)

На бэкенде можно временно добавить незащищённый POST-эндпоинт (например `/api/setup/create-admin`), который один раз создаёт пользователя с `is_admin = true` и затем отключить или удалить этот эндпоинт.

---

## Проверка

1. Откройте фронтенд: `/admin/login`.
2. Введите **Email:** `kana.bahytzhan@gmail.com`, **Пароль:** `Kanaezz15!`.
3. После успешного входа должно произойти перенаправление на `/admin/users`.

Если вход не удаётся — проверьте на бэкенде: что пользователь есть в БД, у него `is_admin = true`, пароль при создании был захэширован тем же алгоритмом, что и при логине.
